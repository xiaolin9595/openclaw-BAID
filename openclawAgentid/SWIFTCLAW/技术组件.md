### PydanticAI + Anthropic Claude 配置示例

```python
from pydantic_ai import Agent
from pydantic_ai.models.anthropic import AnthropicModel

# Anthropic Claude 配置（推荐）
model = AnthropicModel(
    'claude-3-5-sonnet-20241022',
    api_key='your-anthropic-api-key'
)

agent = Agent(model)

# 运行 Agent
result = await agent.run('Hello!')
print(result.data)
```

### 带工具的 Agent 示例

```python
from pydantic_ai import Agent, RunContext

@agent.tool
def search(ctx: RunContext, query: str) -> str:
    """搜索工具"""
    return f"搜索结果: {query}"

result = await agent.run('搜索 Python 教程')
```

## 开发模式：SDD + TDD

### Spec-Driven Development (SDD)

使用 `superpowers` 插件进行规格驱动开发，核心流程：

```
需求/想法
    ↓
调用 Skill（brainstorming）
    ↓
生成结构化规格文档（Spec）
    ↓
调用 Skill（writing-plans）
    ↓
编写实现计划（Plan）
    ↓
调用 Skill（test-driven-development）
    ↓
编写测试 → 编写实现 → 运行验证
    ↓
调用 Skill（verification-before-completion）
    ↓
确认完成
```

**关键 Skill**：
- `superpowers:brainstorming` - 任何创造性工作前的头脑风暴
- `superpowers:test-driven-development` - 测试驱动开发
- `superpowers:verification-before-completion` - 完成前的验证
- `superpowers:writing-plans` - 多步骤任务的计划编写
- `superpowers:executing-plans` - 执行计划

### Test-Driven Development (TDD)

遵循 superpowers TDD 规范：

1. **红**：编写失败的测试
2. **绿**：编写最小实现让测试通过
3. **重构**：优化代码，保持测试通过

**TDD 规则**：
- 每个功能必须有对应的测试
- 实现代码必须在测试之后编写
- 只编写让测试通过的最小代码
